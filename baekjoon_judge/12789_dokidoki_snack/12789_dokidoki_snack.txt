12789 도키도키 간식드리미

일단 장소가 너무 헷갈리게 설명되어 있는데, 편하게 영문표기로 변경하겠습니다.
처음 -> A
쉬어가는곳 -> B
도착점 -> C

그림을 보면 3가지 이동이 있다는걸 알 수 있죠.
1. A -> C
2. B -> C
3. A -> B

네. 그렇습니다. 해보면 됩니다.

1과 2가 되는지 확인해보고, 안되면 3입니다.
중간에 1,2,3 전부 안되는 부분이 생기면 Sad 그외는 Nice


Tip) 위 로직을 그냥 배열가지고 놀아도 상관은 없는데 코딩하기 어렵잖아요? 그러니까 우리 자료구조를 한번 써봅시다.
B는 마지막에 들어간 녀석이 제일 빨리 나옵니다. 이거 되게 익숙하지 않나요? Last in First out, Stack입니다. 같은 원리로 A도 Stack화 시킬 수 있습니다. 얘는 pop만 가능해야겠죠.
그럼 3가지 이동을 다시 표시해보면
1. A.pop()
2. B.pop()
3. B.push(A.pop())

요렇게 됩니다. 간단하죠.
